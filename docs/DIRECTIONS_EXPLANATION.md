# Направления (Directions) в системе

## Что такое направление?

**Направление** (direction) - это область развития, по которой пользователь проходит тестирование компетенций. 

Примеры направлений:
- Backend Development
- Frontend Development  
- Data Science
- DevOps
- Mobile Development

## Как это работает?

1. **Пользователь выбирает направление** → например, "Backend Development"
2. **Создается assessment** → `POST /api/assessments/{direction_id}`
3. **Система получает компетенции** для этого направления
4. **Проводится тестирование** по компетенциям
5. **Формируется roadmap** для выбранного направления на основе результатов

## Технические детали

### В базе данных

- Направление хранится в таблице `roles`
- Каждое направление имеет набор `competencies` (компетенций)
- Компетенции связаны с направлением через `role_id` в таблице `competencies`

### В API

- Для получения списка направлений: `GET /api/roles`
- Для создания тестирования: `POST /api/assessments/{direction_id}`
- Параметр называется `direction_id`, но в БД это `role_id`

### Почему "направление", а не "роль"?

- **"Направление"** - более понятное название для пользователя
- Отражает суть: это область, в которой пользователь хочет развиваться
- **"Роль"** - технический термин, который может быть непонятен

## Пример использования

```typescript
// 1. Получить список доступных направлений
const directions = await fetch('/api/roles', {
  headers: { 'Authorization': `Bearer ${userId}` }
}).then(r => r.json());

// 2. Пользователь выбирает направление (например, "Backend Development")
const selectedDirection = directions.roles[0]; // direction_id = selectedDirection.id

// 3. Создать тестирование для выбранного направления
const assessment = await fetch(`/api/assessments/${selectedDirection.id}`, {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${userId}` }
}).then(r => r.json());

// 4. По результатам тестирования будет сформирован roadmap
// для выбранного направления
```

## Связь с Roadmap

После завершения тестирования система автоматически:
1. Анализирует результаты по всем компетенциям направления
2. Выявляет пробелы в знаниях
3. Генерирует персональный roadmap для развития в выбранном направлении
4. Roadmap включает материалы, задачи и вопросы для каждой компетенции
